A word about CSS Animations:

Many resources were provided to us by our instructors for our journey to start out with web development. Few of them stuck out to me as much as a presentation by Vanessa Alvarez, an expert on animations who is passionate about them. Her presentation was mainly on “why?” animation might be chosen or not chosen for a given project. She starts off with the caveat that it shouldn’t be overused, an important pitfall of animation as a tool.

What it is great for is that it breathes life, and with that, emotion, into an inanimate webpage (forgive the pun). It has the ability to start a “conversation” between page and user. With the help of animations, pages can “talk” to users by signaling, yes the page is loading, yes the information you have input is being sent. It also can let a user know, no the information you input was wrong/incompatible such as when filling out forms. It can also direct a user’s eyes to the important buttons on the page to deepen the relationship between a user and the page. Finally, they can simply set the mood of the page or brand of the page. Animations can be made to be sleek, cool, futuristic, snappy, slow, sleepy, calm, and everything in between.

Ultimately, most animations are resource heavy and require the knowhow of advanced technologies and graphical design knowledge. However, one understated aspect of regular old CSS in basic webpages is that it too can be used to animate. As such, it is easy to use without any knowledge of JavaScript required. It is light on hardware resources trying to load a page and therefore quite reliable across many if not all devices, and to build upon that, because browsers handle all CSS rendering it lets browsers optimize for their performance and efficiency.
